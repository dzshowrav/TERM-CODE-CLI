# Verification Before Completion

Do not say "done" until you have fresh evidence.

## Verify

- the intended files changed
- the relevant tests or checks passed
- the output matches the request
- the docs are updated if the interface changed
- the receipt does not overstate runtime behavior

## Output

Summarize:

- what changed
- what was verified
- what remains unknown
